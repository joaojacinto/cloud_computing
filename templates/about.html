About / Help

Cloud Images App is an academic project for the Cloud Computing course. It allows users to upload, analyze, and manage images using Google Cloud services, providing a simple example of a real cloud-native web application.

Cloud Services Used

  • Google Cloud Run: Scalable deployment of the containerized Flask app.
  • Google Cloud Storage: Storage of all uploaded images (with direct download links).
  • Google Cloud Firestore: NoSQL database for metadata, labels, and additional info.
  • Google Cloud Vision API: Automatic image analysis (object detection & labeling).

How It Works

  1. The user uploads an image via the web interface.
  2. The image is stored in Cloud Storage.
  3. The Vision API analyzes the image and extracts labels with confidence scores.
  4. Metadata and results are stored in Firestore.
  5. The dashboard allows viewing, downloading, and deleting images, with details.

Features

  • Modern, responsive UI (Bootstrap 5 + SweetAlert2)
  • Upload with live image preview & progress bar
  • Dashboard with pagination and card previews
  • Download and delete images (with confirmation)
  • Label details, upload time, and file size shown

Architecture (summary)

  • Frontend: Flask, HTML, Bootstrap, SweetAlert2, Bootstrap Icons
  • Backend: Python Flask, Google Cloud SDKs (Storage, Vision, Firestore)
  • Infra: Docker container running on Google Cloud Run

Help / Support

  • If you encounter errors, check your internet connection.
  • Ensure your GCP service account has the correct permissions.
  • For technical questions, check Google Cloud documentation or contact the project author.
