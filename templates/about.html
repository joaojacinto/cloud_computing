About / Help

Cloud Images App is an academic project for the Cloud Computing course. It allows users to upload, analyze, and manage images using Google Cloud services.

Cloud Services Used:

  • Google Cloud Run: Runs the containerized Flask app, providing scalable deployment without managing servers.
  • Google Cloud Storage: Stores all uploaded images, with public direct links for downloading.
  • Google Cloud Firestore: NoSQL database to store image metadata, labels, and additional info.
  • Google Cloud Vision API: Automatically analyzes images to detect objects and generate descriptive labels.

How It Works:

  1. The user uploads an image through the web interface.
  2. The image is stored in Cloud Storage.
  3. Automatic analysis is performed using the Cloud Vision API (labels + confidence).
  4. Results and metadata (name, date, size, labels, etc.) are stored in Firestore.
  5. The dashboard shows all images, allows downloads, displays details, and enables record deletion.

Features:

  • Image upload with automatic analysis
  • Dashboard with details (labels, confidence, date, size, etc.)
  • Direct image download
  • Pagination (for large image collections)
  • Delete images from both Storage and Firestore
  • Visual feedback for success/errors (SweetAlert2)

Architecture (summary):

  • Frontend: Flask + HTML/CSS/Bootstrap + SweetAlert2
  • Backend: Python Flask, Google Cloud SDKs
  • Infrastructure: Docker container on Google Cloud Run

Help / Support:

  • If you encounter errors, ensure you have a working internet connection.
  • Make sure your GCP service account permissions are correctly set up.
  • For technical questions, consult the Google Cloud documentation or contact the project owner.
